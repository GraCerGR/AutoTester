selenium grid как доставать результаты тестов из узлов:
Результаты тестов из узлов Selenium Grid (Hub и Nodes) можно получать с помощью WebDriver или REST API. Это зависит от задачи: нужно проверить результаты тестов, которые выполняются на узлах, или получить информацию о конфигурации хаба (Hub).

С помощью WebDriver
Результаты тестов, которые выполняются на узлах, передаются обратно в хаб. Это происходит по следующему алгоритму: 
blog.apify.com
Скрипт теста (клиент) запрашивает новую сессию у хаба, указывая желаемый браузер и платформу.
Хаб проверяет реестр доступных узлов и направляет запрос теста на подходящий узел, способный выполнить желаемые возможности.
Узлы запускают указанный браузер с желаемой конфигурацией и устанавливают новую сессию WebDriver.
Скрипт теста общается с браузером через сессию WebDriver на узле для выполнения теста.
Результаты тестов и статус передаются обратно в хаб, который передаёт их клиенту.
 
blog.apify.com
Важно: каждый узел в Selenium Grid должен иметь соответствующий драйвер веб-браузера. Например, если узел настроен на запуск тестов на Chrome, на нём должен быть установлен ChromeDriver.

С помощью REST API
В Selenium Grid есть API, который позволяет получать информацию о конфигурации хаба. Например: 
galilsoftware.com
stackoverflow.com
Команда /grid/api/hub/ — возвращает JSON-данные о конфигурации хаба, включая количество доступных слотов и количество пустых слотов. Это важно для проверки конфигураций для больших Selenium Grid, которые были созданы с помощью автоматических скриптов.
Команда /grid/api/testsession?session=<SessionId> — возвращает информацию о том, к какому узлу хаб направил запрос теста. ID сеанса можно получить с помощью driver.getSessionId() (драйвер — типа RemoteWebDriver или его подклассы).

stackoverflow.com
